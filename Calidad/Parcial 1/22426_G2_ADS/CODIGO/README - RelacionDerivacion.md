# Principio de Derivación en el Proyecto

## Descripción

La relación de **Derivación** indica que un Elemento de Configuración del Software (ECS) se origina a partir de otro elemento previo. Un ejemplo clásico es cuando se genera código objeto a partir del código fuente, o cuando se obtiene una traza de ejecución derivada de una prueba aplicada sobre un programa compilado.

En nuestro proyecto, esta relación ha sido fundamental para asegurar la trazabilidad y el control de versiones del código fuente.

## Aplicación en el Proyecto

Para la elaboración de la versión final del código fuente (**CF_V1.0.2**), nos basamos en una línea base previa establecida en la carpeta:

[PREGAME - Línea Base Código Proyecto V1.0.0](https://github.com/DylanGReinosoT/22426_G2_ADS/tree/main/PREGAME/1.%20ELICITACION/1.0%20Linea%20Base/LBC/CodigoProyecto_V1.0.0)

Esta fue una de nuestras líneas base que sirvió para llegar a la versión final del código:

[CODIGO - CF_V1.0.2](https://github.com/DylanGReinosoT/22426_G2_ADS/tree/main/CODIGO/CF_V1.0.2)

Así, garantizamos que cada cambio y evolución del código pueda ser rastreado hasta su origen, manteniendo un control claro y efectivo durante todo el proceso de desarrollo.
